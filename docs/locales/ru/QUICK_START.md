# Краткое руководство по squid

Для конечного пользователя: локальная установка и запуск squid, настройка модели и начало диалога и задач. Архитектура и расширения — в [developer-guide.md](./developer-guide.md).

## Назначение продукта

**squid** — **локальный AI‑рабочий стол** для:

- чтения кода, ревью и лёгких правок в заданной рабочей области (в рамках режима задачи и песочницы);
- задач с навыками и экспертами: документы, поиск, структурированный вывод;
- локальных cron‑задач с вызовом модели по расписанию;
- опциональных **расширений каналов** Feishu, Telegram, WeChat и др. (отдельная настройка, см. [channel-extensions.md](./channel-extensions.md)).

## Окружение и запуск

**Из исходников (для разработчиков)**

- Node.js 22 LTS и npm; десктоп использует Electrobun (при `npm run dev` CLI подготавливается под платформу).
- В корне проекта должен быть **`electrobun.config.ts`** (Electrobun читает только это имя файла).

```bash
cd squid
npm install
npm run dev
```

**Готовые сборки**

См. релизы GitHub и инструкции для вашей ОС. На macOS неподписанные сборки могут требовать «Открыть через контекстное меню» или разрешения в настройках безопасности; подробнее в [README.md](../../README.md) в корне репозитория.

## API‑ключи

Настройте хотя бы одного провайдера модели в **настройках** приложения (ключи пишутся в `~/.squid/config.json` на этой машине):

| Провайдер | Примечание |
|-----------|------------|
| Anthropic | Ключ в [Anthropic Console](https://console.anthropic.com/) |
| OpenAI | Ключ на [OpenAI Platform](https://platform.openai.com/) |
| Совместимый эндпоинт | Свой Base URL и имя модели (протокол должен быть совместим с тем, что ожидает приложение) |

## Первый запуск

1. После старта откройте боковую панель **«Настройки»** и сохраните параметры модели.
2. В чате или задаче **выберите рабочий каталог** (не используйте недоверенный корень).
3. **Создайте сессию или задачу**, выберите режим:
   - **Ask**: преимущественно чтение и анализ, без активной перезаписи файлов (уточняйте по версии).
   - **Craft**: разрешена цепочка инструментов, возможны изменения в рабочей области.
   - **Plan**: планирование и пошаговые инструкции для сложных задач.
4. При необходимости подключите **skill** или **эксперта**.

## Каналы и Feishu (опционально)

- В разделе **«Каналы»** виден встроенный WebUI и статус расширений.
- Реализация Feishu в репозитории: `extensions/feishu/`; включение в `config/channel-extensions.json` зависит от репозитория. Пользовательский список: `~/.squid/channel-extensions.json`.
- Сторонние расширения: `~/.squid/extensions/<каталог>/`, подробнее [channel-extensions.md](./channel-extensions.md).

Создание бота Feishu, long polling / webhook и поля `~/.squid/feishu-channel.json` — по текстам в приложении и [user-guide.md](./user-guide.md).

## Примеры задач

**Ревью кода (Ask)**

```text
Режим: Ask
Рабочий каталог: <путь к проекту>
Запрос: кратко опиши назначение основных модулей в src и дай замечания по читаемости и явным дефектам.
```

**Документация пакетом (Craft)**

```text
Режим: Craft
Рабочий каталог: <путь к проекту>
Запрос: подготовь черновик Markdown для публичных API в указанном каталоге.
```

**Cron**

На странице **«Планировщик»** создайте запись: Cron и текст, который после срабатывания передаётся модели; если приложение не запущено, расписание не выполняется.

## Навыки и эксперты

- **Skills**: выбор в UI; файлы в `~/.squid/skills/` (в т.ч. установки из SkillHub).
- **Эксперты**: роль и границы ответов; управление на соответствующих страницах.

## Частые вопросы

**Ключи только локально?**  
Да. Не коммитьте `~/.squid` в Git; делайте резервные копии.

**Задачи меняют файлы?**  
Зависит от режима и политики инструментов: Ask — преимущественно чтение; Craft — возможна запись; Plan — обычно сначала план. Ориентируйтесь на подсказки UI.

**Границы рабочего каталога?**  
Файловые инструменты обычно ограничены привязанным к сессии каталогом; детали — в реализации прав и песочницы.

**Как остановить выполняющуюся задачу?**  
Кнопка остановки/прерывания в UI задачи или сессии (название может отличаться).

**Cron после закрытия приложения?**  
Нет; планировщик привязан к работающему процессу.
