# Обзор проекта squid

Кратко описывает границы возможностей репозитория и разбиение на модули для продуктовой и технической ревизии; при расхождении с кодом приоритет у исходников.

## Позиционирование

**squid** — AI‑рабочий стол с приоритетом локального запуска (Electrobun + Bun + системный WebView). Данные по умолчанию: `~/.squid`.

## Реализованные возможности (кратко)

### Задачи и контекст

- Модель данных задач и конечный автомат Ask / Craft / Plan  
- Сжатие контекста и персистентность задач  
- Права и классификация рисков инструментов  

### Модели

- Адаптеры Anthropic, OpenAI, DeepSeek и др. с реестром (см. `src/models`)  
- Потоковый вывод, подсчёт токенов, шифрованное хранение ключей  

### Рабочая область и инструменты

- Привязка рабочего каталога, песочница путей  
- ReadFile, WriteFile, Glob, Grep и единое отображение результатов с лимитом размера  

### Навыки и эксперты

- YAML skills, загрузчик, белый список и hooks  
- Встроенные skills и шаблоны экспертов; часть UI может эволюционировать  

### Claw и планирование

- HTTP‑сервис Claw и обработка задач (`src/claw`); включение по умолчанию в десктопе — см. `src/bun/index.ts`  
- Cron на node-cron, история запусков, почтовые уведомления (если настроены)  

### Каналы

- Реестр каналов, встроенный WebUI  
- Расширения: `extensions/` + пользовательский каталог, декларативный manifest и мост TaskAPI  
- EventBridge, WebSocket и связь с UI (см. `docs/webui-channel.md` и др.)  

### Десктоп и фронтенд

- React: основной layout, настройки, задачи и сессии  
- Локальный HTTP API в главном процессе (`Bun.serve`) для UI  

### Качество

- Модульные и интеграционные тесты Vitest (см. [TEST_REPORT.md](./TEST_REPORT.md))  
- Пользовательская и разработческая документация в `docs/`  

## Тесты и контроль качества

Последняя зафиксированная выборка: 9 файлов тестов, 31 кейс — зелёные (см. TEST_REPORT). Перед слиянием рекомендуется локально выполнить `npm test`.

## Безопасность (кратко)

- Песочница путей и флаги read-only / destructive у инструментов  
- Локальная защита ключей AES-256-GCM (см. реализацию `secure-storage`)  
- Токены Claw и движок прав (если включены соответствующие пути)  

## Производительность (кратко)

- LRU, виртуальный скролл, ленивая загрузка, потоковые ответы, сжатие контекста — по конкретным модулям
