# squid v0.1.0 — заметки о выпуске

## Недавние изменения (2026-04-10)

### Цепочка распознавания изображений в каналах

- Входящие изображения из Telegram / Feishu / личного WeChat унифицированы: загрузка в workspace + `mentions(file)`.
- При постановке в очередь из-за занятости канала сохраняется тот же `mentions`, чтобы картинка не терялась в пути очереди.
- Общая для расширений запись на диск: `extensions/shared/workspace-image-store.ts`.

### Команда прерывания канала `/wtf`

- В `TaskAPI.executeTaskStream` добавлена команда `/wtf`, по смыслу как ESC в Web: прерывается только текущая задача сессии, очередь не очищается.
- Ветка `/wtf` выполняется **до** проверки «сессия занята», чтобы прерывание срабатывало сразу, а не превращалось в busy.
- Мосты Telegram / Feishu / WeChat дополнены тестами на прохождение `/wtf` в общую ветку команд.

## Обзор

Первый публичный выпуск **squid**: локальный AI‑рабочий стол на Electrobun с мультимодельным чатом, режимами задач, skills и экспертами, cron и расширяемыми каналами (Feishu / Telegram / WeChat и др. — по включению).

## Основные возможности

### Задачи и рабочая область

- Режимы: Ask (чтение), Craft (автоинструменты), Plan (планирование)
- Конечный автомат и персистентность задач
- Привязка рабочего каталога и песочница путей

### Модели

- Серия Claude Anthropic (доступные имена — в настройках)
- Совместимый с OpenAI API
- DeepSeek и др. совместимые эндпоинты (зависит от адаптеров и настроек)
- Потоковый вывод и подсчёт токенов (по фактической реализации)
- Локальное шифрованное хранение ключей API

### Навыки и эксперты

- Встроенные шаблоны skills; загрузка из `~/.squid/skills` и установка из SkillHub и др.
- Встроенные роли экспертов и точки расширения

### Каналы

- Встроенный канал WebUI
- Расширения: `extensions/` и `~/.squid/extensions`, декларативная конфигурация и мост TaskAPI

### Claw и автоматизация

- HTTP‑возможности Claw и токены — см. `src/claw`; включение сервиса Claw в десктопе по умолчанию — см. `src/bun/index.ts`
- Cron на node-cron и история запусков

### Десктоп

- Electrobun: главный процесс Bun + системный WebView
- Основной layout, настройки, UI задач и сессий

## Тесты

Последняя зафиксированная автоматизация: 9 файлов, 31 тест — успешно (см. [TEST_REPORT.md](./TEST_REPORT.md)). Перед релизом выполните `npm test` на целевой среде.

## Установка и команды (исходники)

```bash
git clone <repository-url>
cd squid
npm install
npm test          # опционально
npm run dev       # разработка десктопа
npm run build     # tsc
npm run build:electron:release   # стабильный канал, артефакты в artifacts/
```

## Конфигурация

Первый запуск: **настройки** → ключи модели → сохранить. Каналы и Feishu: [QUICK_START.md](./QUICK_START.md), [channel-extensions.md](./channel-extensions.md).

**Сборка**: Electrobun читает **только `electrobun.config.ts`**; без файла или при ошибочном `.js` в stable‑сборке может не скопироваться `public` и UI останется белым экраном.
