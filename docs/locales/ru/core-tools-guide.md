# Руководство по основным инструментам

Описание десяти ключевых инструментов squid: назначение, параметры и ограничения.

## 1. FileEditTool (file_edit)

**Назначение**: точечная замена фрагмента в файле по точному совпадению строки.

**Входные параметры:**

- `file_path` (string): путь к файлу
- `old_string` (string): искомый фрагмент
- `new_string` (string): текст замены
- `replace_all` (boolean, опционально): заменить все вхождения (по умолчанию false)

**Пример:**

```typescript
{
  file_path: "src/index.ts",
  old_string: "const port = 3000",
  new_string: "const port = 8080"
}
```

**Ограничения:**

- Несколько совпадений без `replace_all=true` — ошибка
- Только точное совпадение строк, без регулярных выражений

---

## 2. BashTool (bash)

**Назначение**: выполнение команд Bash с таймаутом и опцией фона.

**Входные параметры:**

- `command` (string): команда Bash
- `working_directory` (string, опционально): рабочий каталог
- `timeout` (number, опционально): таймаут в мс, по умолчанию 30000
- `run_in_background` (boolean, опционально): запуск в фоне

**Пример:**

```typescript
{
  command: "npm install",
  working_directory: "/path/to/project",
  timeout: 60000
}
```

**Ограничения:**

- Нет интерактивного ввода
- Фоновые задачи не переживают перезапуск
- Помечен как деструктивный — нужно доверие пользователя

---

## 3. PowerShellTool (powershell)

**Назначение**: выполнение команд PowerShell (**только Windows**).

**Входные параметры:**

- `command` (string): команда PowerShell
- `working_directory` (string, опционально)
- `timeout` (number, опционально), по умолчанию 30000
- `run_in_background` (boolean, опционально)

**Пример:**

```typescript
{
  command: "Get-Process | Where-Object {$_.CPU -gt 100}",
  timeout: 10000
}
```

**Ограничения:**

- Только Windows; на других ОС — ошибка

---

## 4. WebSearchTool (web_search)

**Назначение**: поиск в интернете через DuckDuckGo, список результатов.

**Входные параметры:**

- `query` (string): поисковый запрос
- `max_results` (number, опционально): максимум результатов (по умолчанию 10, не больше 10)

**Пример:**

```typescript
{
  query: "TypeScript best practices",
  max_results: 5
}
```

**Ограничения:**

- Зависимость от HTML‑разметки DuckDuckGo — при изменении сайта возможны сбои
- Без API‑ключа; качество может уступать платным API
- Не более 10 результатов

---

## 5. Группа инструментов Cron

### 5.1 CronCreateTool (cron_create)

**Назначение**: создать отложенную по cron задачу.

**Параметры:**

- `cron_expression` (string): выражение cron (например `"0 * * * *"` — каждый час)
- `task_content` (string): описание содержимого задачи

**Пример:**

```typescript
{
  cron_expression: "0 9 * * *",
  task_content: "Ежедневный бэкап в 9:00"
}
```

### 5.2 CronDeleteTool (cron_delete)

**Назначение**: удалить задачу по id.

**Параметры:**

- `task_id` (string)

### 5.3 CronListTool (cron_list)

**Назначение**: список всех cron‑задач.

**Параметры:** нет

**Ограничения:**

- Хранение в памяти — после перезапуска теряется
- Персистентность может появиться в следующих версиях

---

## 6. SkillTool (skill)

**Назначение**: вызов зарегистрированного skill (шаблон задачи).

**Параметры:**

- `skill_name` (string): имя skill
- `args` (string, опционально): аргументы

**Пример:**

```typescript
{
  skill_name: "code-review",
  args: "src/components/Button.tsx"
}
```

**Ограничения:**

- Только skills с `user-invocable: true`
- Файлы skills — в `~/.squid/skills/`
- Нужна настроенная модель (`~/.squid/config.json`)
- Выполнение идёт по общему конвейеру и может вызывать инструменты

---

## 7. BriefTool (brief)

**Назначение**: краткое или развёрнутое резюме текста.

**Параметры:**

- `content` (string): исходный текст
- `prompt` (string, опционально): свой промпт
- `type` (enum, опционально): `brief` | `detailed` | `bullet_points`

**Пример:**

```typescript
{
  content: "Длинный текст...",
  type: "bullet_points"
}
```

**Ограничения:**

- Требуется `ANTHROPIC_API_KEY` в окружении
- Обрезка при длине > 50000 символов
- Вызов внешнего API — возможны расходы

---

## 8. AgentTool (agent)

**Назначение**: дочерний агент со своим контекстом для сложных задач.

**Параметры:**

- `instruction` (string): инструкция
- `timeout` (number, опционально): мс, по умолчанию 300000 (5 минут)

**Пример:**

```typescript
{
  instruction: "Проанализируй все TypeScript-файлы проекта на предмет узких мест по производительности",
  timeout: 600000
}
```

**Ограничения:**

- Нужна конфигурация модели
- Таймаут по умолчанию 5 минут, настраивается
- Общий исполнитель возвращает структурированные метаданные (исполнитель, режим, рабочий каталог, длительность)

---

## Атрибуты инструментов

У каждого инструмента:

- **isConcurrencySafe**: безопасен ли для параллельного запуска
- **isReadOnly**: только чтение
- **isDestructive**: может менять состояние системы

## Персистентность результатов

Все инструменты реализуют `mapToolResultToToolResultBlockParam`:

- при превышении `maxResultSizeChars` результат сохраняется на диск
- в контекст возвращается превью

## Безопасность

1. **BashTool / PowerShellTool** — произвольные команды ОС  
2. **FileEditTool** — прямая правка файлов; желателен контроль версий  
3. **BriefTool / AgentTool** — внешние API, берегите ключи  
4. **WebSearchTool** — проверяйте содержимое страниц на вредоносное
