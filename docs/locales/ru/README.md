# squid

**squid** — локальный AI‑рабочий стол: в окне приложения вы общаетесь с моделью, управляете несколькими сессиями, задаёте рабочий каталог для каждой сессии и в настройках подключаете Anthropic, OpenAI или совместимые эндпоинты. Навыки (skills), память, задачи по расписанию, а также каналы вроде Feishu, Telegram, WeChat и др. (включаются и настраиваются в приложении) работают в общем конвейере задач. Конфигурация и данные по умолчанию хранятся в **`~/.squid`** на вашей машине.

**Версия**: 0.1.0  
**Лицензия**: MIT

---

## Возможности

- **Несколько чат‑сессий**: диалоги по потокам, привязка рабочего каталога к сессии — удобно задавать вопросы по конкретному проекту или разрешать ассистенту читать/писать файлы в допустимых границах.
- **Режимы задач**: в интерфейсе выбирайте **Ask** (консультации и преимущественно чтение), **Craft** (цепочка инструментов с выполнением), **Plan** (планирование и декомпозиция); точное поведение см. в подсказках приложения.
- **Модели и ключи**: в **настройках** указываются API‑ключ, имя модели и при необходимости свой base URL; ключи хранятся только локально и не распространяются вместе с репозиторием.
- **Навыки**: просмотр, установка и управление skills (в т.ч. из Tencent SkillHub); установленные файлы лежат в `~/.squid/skills`.
- **Эксперты и память**: предустановленные или свои «эксперты» для стиля и границ ответов; долговременная память доступна для просмотра и обслуживания.
- **Cron‑задачи**: по выражению Cron локально запускается передача заданного текста модели, с историей выполнения.
- **Каналы**: помимо основного UI — опциональные каналы Feishu, Telegram, личный WeChat и др. (настройка в разделе каналов; часть каналов требует входа или webhook — см. `docs` и описания расширений).

## Установка и запуск

**Из исходников (разработчики и сборка сами)**

- Нужны **Node.js** (рекомендуется 22 LTS) и **npm**; для десктопа используется **Electrobun**; поддерживаются macOS 14+, Windows 11+ и Linux в объёме, заявленном в документации Electrobun.
- После клонирования репозитория в корне проекта:

```bash
cd squid
npm install
npm run dev
```

**Готовые сборки**

- Если в релизах (например GitHub Releases) есть артефакты под вашу ОС — установите или распакуйте по инструкции; на macOS неподписанные/ненотаризованные сборки могут блокироваться Gatekeeper — при необходимости разрешите в «Конфиденциальность и безопасность».

## Первые шаги

1. Откройте **настройки**, настройте модель и при необходимости каналы, сохраните.  
2. В чате **выберите рабочий каталог** (не используйте недоверенные пути как корень рабочей области).  
3. **Создайте сессию** и коротко проверьте сценарий; затем подключайте skills, cron и каналы по мере необходимости.

Подробнее: **[QUICK_START.md](./QUICK_START.md)** и **[user-guide.md](./user-guide.md)**.

## Где хранятся данные

| Путь | Назначение |
|------|------------|
| `~/.squid/config.json` | Основной конфиг: ключи моделей, UI и часть флагов функций |
| `~/.squid/skills/` | Установленные навыки |
| Прочие JSON в `~/.squid` | Конфиги и данные каналов, памяти и др. (появляются по мере использования) |

Делайте резервные копии каталога; не публикуйте ключи в открытых репозиториях. Для расширения личного WeChat при разработке из исходников может понадобиться **`npm run weixin-personal:login`** — см. документацию расширения.

## Безопасность

- Если у ассистента есть доступ к файлам или командам, он ограничен **рабочим каталогом** и встроенными правилами; не выбирайте системные чувствительные каталоги по умолчанию.  
- Приложение поднимает локальный сервис для связи UI и главного процесса; в штатном режиме он не предназначен для доступа из LAN/интернета — при пробросе портов или reverse proxy настройте аутентификацию и контроль доступа сами.

## Разработка из исходников (кратко)

squid построен на **Electrobun**: главный процесс и локальный сервис на стороне Bun, интерфейс в системном WebView. Если вы разрабатываете **в корне клонированного репозитория** и хотите подхватить встроенные расширения каналов, задайте **`SQUID_ROOT`** на корень репозитория (чтобы находился `config/channel-extensions.json`); пользователям установочных пакетов это обычно не нужно. Архитектура, расширения и инструменты: **[developer-guide.md](./developer-guide.md)** и **[tool-development-guide.md](./tool-development-guide.md)**.
