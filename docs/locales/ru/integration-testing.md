# Руководство по интеграционному тестированию

Как проверять функциональность **openclaw-compatible-channels**.

## Предварительные условия

1. Зависимости: `npm install`  
2. Настроенный API Key (страница настроек)  
3. Запущенное приложение: `npm run dev`  

## Сценарии

### Сценарий 1: уведомление о завершении cron в чате

**Цель:** после cron задача завершилась — в чате есть уведомление.

**Шаги:**

1. Запуск  
   ```bash
   npm run dev
   ```

2. DevTools браузера → Console  

3. Проверка WebSocket  
   Ожидается лог вроде: `[WebSocket] connected` / успешное соединение (точная строка зависит от сборки).

4. Создать cron‑задачу (инструмент cron или API)  
   ```typescript
   const result = cronManager.createTask('*/1 * * * *', 'print current time');
   console.log(result);
   ```

5. Подождать до 1 минуты  

6. В чате должно появиться уведомление с id задачи, статусом, длительностью.

**Ожидание:** соединение установлено, задача выполнена, уведомление с полным payload.

---

### Сценарий 2: уведомление о фоновой задаче

**Цель:** не cron, обычная задача — по завершении уведомление в чате.

**Шаги:**

1. Ввести в чат, например:  
   `Напиши программу Hello World`

2. Отправить и дождаться конца выполнения  

3. Проверить уведомление с результатом  

**Ожидание:** задача завершена, уведомление отображается.

---

### Сценарий 3: команда из чата в движок

**Цель:** команда с фронта доходит до сервера.

**Шаги:**

1. Console DevTools  

2.  
   ```javascript
   window.wsClient.sendCommand('test-command', { param: 'value' });
   ```

3. В логах сервера — приём команды и событие command в EventBridge.

**Ожидание:** отправка успешна, сервер получил, EventBridge сработал.

---

### Сценарий 4: автоматическое переподключение WebSocket

**Цель:** после обрыва клиент снова подключается.

**Шаги:**

1. Убедиться, что WebSocket подключён  

2. Остановить бэкенд (имитация обрыва)  

3. В Console — сообщения о закрытии и попытках реконнекта с увеличивающейся задержкой  

4. Снова запустить сервис  

5. Соединение восстановлено  

**Ожидание:** обрыв замечен, реконнект, успешное восстановление.

---

### Сценарий 5: несколько клиентов

**Цель:** два вкладки браузера получают одни и те же broadcast‑уведомления.

**Шаги:**

1. Две вкладки с приложением  
2. Завершить задачу в одной  
3. Обе должны получить уведомление  

**Ожидание:** два активных клиента, оба получили событие, в логах сервера — два соединения.

---

### Сценарий 6: плагин Feishu OpenClaw (нужны учётные данные Feishu)

**Цель:** сообщения ходят через плагин Feishu.

**Предусловия:** установлен плагин OpenClaw Feishu, заданы appId и appSecret.

**Шаги:**

1. Загрузка плагина  
   ```typescript
   import { createOpenClawAdapter } from './channels/openclaw-adapter/adapter';
   import feishuPlugin from '@openclaw/feishu-plugin';
   
   const adapter = createOpenClawAdapter(feishuPlugin, 'feishu');
   channelRegistry.register(adapter);
   await adapter.setup.initialize();
   ```

2. Конфигурация  
   ```typescript
   adapter.config.set('appId', 'your-app-id');
   adapter.config.set('appSecret', 'your-app-secret');
   ```

3. Завершить задачу — проверить сообщение в Feishu  
4. Отправить сообщение из Feishu — проверить приём в приложении  

**Ожидание:** плагин инициализирован, уведомления уходят и приходят.

## Устранение неполадок

### WebSocket не подключается

**Симптом:** ошибка соединения в Console.

**Проверить:** процесс запущен, порт 8080 свободен, файрвол не режет localhost.

**Действия:**

```bash
lsof -i :8080
```

При необходимости сменить порт в `config/channels.json`.

### Нет уведомления о задаче

**Проверить:** WebSocket активен, вызывается `notifyTaskComplete`, нет ошибок в Console.

**Диагностика:**

```javascript
console.log(window.wsClient.isConnected());
```

### Плагин Feishu не стартует

**Проверить:** установка пакета, полнота конфигурации, доступность API Feishu.

```bash
npm install @openclaw/feishu-plugin
```
