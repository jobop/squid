# Руководство пользователя squid

Основные возможности десктопного приложения. При расхождении с кодом или UI ориентируйтесь на установленную версию.

## Установка и запуск

**Из исходников**

```bash
cd squid
npm install
npm run dev
```

`npm run build` — компиляция TypeScript; `npm start` → `node dist/main.js` (не основной путь Electrobun). Для десктопа в разработке используйте `npm run dev`.

**Готовые сборки**  
См. [README.md](../../README.md) в корне репозитория (в т.ч. macOS и карантин файлов).

## Настройки

При первом запуске:

1. Боковая панель **«Настройки»**.  
2. Ключи и параметры **Anthropic / OpenAI / совместимого** эндпоинта, при необходимости Base URL.  
3. Сохранить — запись в `~/.squid/config.json`.  

Дополнительно: каналы, внешний вид и др. (зависит от версии UI).

## Задачи и сессии

### Режимы

| Режим | Когда использовать |
|-------|---------------------|
| Ask | Вопросы, анализ с минимальными правками файлов |
| Craft | Автовызов инструментов, возможны изменения в рабочей области |
| Plan | Сложные задачи: сначала план или шаги, затем выполнение по согласованию |

### Типовой сценарий

1. **Новая задача** или аналогичный пункт меню.  
2. Режим, модель, **рабочий каталог** (обязателен и должен быть доверенным).  
3. Опционально: skill, эксперт.  
4. Текст запроса и отправка.  

Пути вне рабочего каталога обычно отклоняются песочницей; не используйте системные каталоги как корень.

## Skills

- Выбор из предустановленных и установленных skills при создании задачи или в настройках.  
- Файлы: `~/.squid/skills/` (поддерживаются каталог и одиночный файл — см. загрузчик).  
- Tencent SkillHub: [tencent-skillhub.md](./tencent-skillhub.md).

## Эксперты

Встроенные шаблоны ролей для стиля и предметной области; управление в разделе экспертов. Кастомизация — по возможностям текущей версии.

## Планировщик (Cron)

1. Страница **«Планировщик»**.  
2. Новая запись: выражение Cron, текст для модели и прочие поля.  
3. Работает **только пока запущено приложение**; после закрытия расписание не выполняется.  

Шаблоны (дайджест, обход репозитория и т.д.) — если есть в UI, выбираются в мастере создания.

## Каналы (Channel)

- **WebUI** — основной чат и задачи, связь с движком по встроенному каналу.  
- **Расширения** — Feishu, Telegram, личный WeChat и др. в `extensions/` и `~/.squid/extensions/`, manifest `channel-plugin.json`; включение и формы: [channel-extensions.md](./channel-extensions.md).

Для Feishu нужны приложение Open Platform, подписка на события (WebSocket или webhook) и локальный `~/.squid/feishu-channel.json`. В режиме HTTP callback URL должен быть достижим с стороны Feishu; подробнее [QUICK_START.md](./QUICK_START.md) и `extensions/feishu`.

## Память

Долговременная память доступна в отдельном интерфейсе; физическое расположение — под `~/.squid` (см. реализацию). В тестах каталог памяти можно переопределить переменными окружения (см. документацию разработчика).

## Claw и локальный API (продвинутый уровень)

- Встроенный **локальный HTTP API** для UI и задач; не выставляйте в интернет без усиления защиты.  
- Реализация в `src/claw`; включён ли Claw по умолчанию при старте десктопа — см. `src/bun/index.ts`. Токены, маршруты и удалённый доступ — по коду и тестам.

## Данные и резервное копирование

| Путь | Содержимое |
|------|------------|
| `~/.squid/config.json` | Основной конфиг и ключи |
| `~/.squid/skills/` | Skills |
| `~/.squid/channel-extensions.json` | Пользовательские включения расширений |
| `~/.squid/extensions/` | Пользовательские расширения |

Резервируйте весь `~/.squid`; не коммитьте секреты.

## Частые вопросы

**Как сменить модель по умолчанию?**  
В настройках или при создании конкретной задачи.

**Можно ли читать файлы вне рабочей области?**  
По умолчанию нет (песочница и права).

**Удаление / перенос?**  
Закройте приложение, скопируйте или удалите `~/.squid`; на новой машине восстановите каталог и установите приложение.
