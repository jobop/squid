# Совместимые с OpenClaw каналы — сводка изменений

## Обзор

Реализована двусторонняя связь на базе EventBridge между движком выполнения и плагинами каналов (включая чат WebUI и плагин Feishu в духе OpenClaw).

## Реализованные возможности

### 1. Шина событий EventBridge

- Простая шина на Node.js EventEmitter
- Уведомления о завершении задач (`notifyTaskComplete`)
- Отправка команд (`sendCommand`)
- Глобальный синглтон, общий для всех модулей
- Изоляция ошибок: сбой подписчика не ломает остальных

**Файлы:**

- `src/channels/bridge/event-bridge.ts`

### 2. Плагин канала WebUI

- WebSocket‑сервер (порт 8080)
- Несколько одновременных клиентов
- Пинг с интервалом 30 с
- Автоматическое переподключение
- Подписка на EventBridge и рассылка всем клиентам
- Приём команд от клиентов и пересылка в EventBridge

**Файлы:**

- `src/channels/plugins/webui/plugin.ts`
- `src/channels/registry.ts`
- `src/channels/index.ts`

### 3. Клиент WebSocket во фронтенде

- Подключение и реконнект с экспоненциальной задержкой
- Отправка heartbeat и ответы на ping
- UI для уведомлений о завершении задач
- Интерфейс отправки команд
- Управление состоянием соединения

**Файлы:**

- `public/websocket-client.js`
- `public/index.html` (интеграция)

### 4. Интеграция с Cron

- При завершении задачи — уведомление через EventBridge
- В payload: сведения о задаче, результат, длительность, статус

**Файлы:**

- `src/tools/cron-manager.ts`

### 5. Интеграция выполнения задач

- Уведомления EventBridge по завершении фоновых задач
- Обработка ошибок и уведомления об отказе

**Файлы:**

- `src/tasks/executor.ts`

### 6. Адаптер плагинов OpenClaw

- Универсальная реализация адаптера
- Отправка/приём сообщений, конфигурация, проверка состояния
- Автоподписка на события EventBridge
- Совместимость с интерфейсом плагинов OpenClaw

**Файлы:**

- `src/channels/openclaw-adapter/adapter.ts`

## Использование

### 1. Запуск приложения

```bash
npm run dev
```

Канал WebUI поднимается автоматически; WebSocket слушает `ws://localhost:8080`.

### 2. Получение уведомлений о задачах

Страница фронтенда сама подключается по WebSocket и показывает уведомления о завершении.

### 3. Отправка команд

```javascript
window.wsClient.sendCommand('restart-task', { taskId: 'task-123' });
```

### 4. Интеграция плагина OpenClaw

```typescript
import { createOpenClawAdapter } from './channels/openclaw-adapter/adapter';
import feishuPlugin from '@openclaw/feishu-plugin';

const adapter = createOpenClawAdapter(feishuPlugin, 'feishu');
channelRegistry.register(adapter);
await adapter.setup.initialize();
```

## Технологии

- **EventBridge**: Node.js EventEmitter (просто и легко)
- **WebSocket**: библиотека `ws`
- **Фронтенд**: нативный WebSocket API
- **Паттерн**: адаптер для совместимости с OpenClaw

## Производительность и надёжность

- **Низкая задержка**: WebSocket в реальном времени
- **Параллельные клиенты**: несколько подключений
- **Отказоустойчивость**: реконнект и изоляция ошибок
- **Расширяемость**: плагинная архитектура

## Известные ограничения

1. **WebSocket только локально** — в текущей версии нет TLS/аутентификации
2. **Адаптер OpenClaw минимален** — реализованы лишь ключевые интерфейсы
3. **Нет персистентности сообщений** — офлайн‑сообщения не сохраняются
