# squid ユーザーガイド

squid デスクトップの主な機能と使い方です。ソースやテストと矛盾する場合は、現行バージョンのコードと画面を正とします。

## インストールと起動

**ソースから**

```bash
cd squid
npm install
npm run dev
```

**注**：`npm run build` は TypeScript コンパイル、`npm start` は `node dist/main.js` であり、Electrobun デスクトップの主経路とは異なります。日常のデスクトップ開発は `npm run dev` を使ってください。

**配布パッケージ**  
各プラットフォームの成果物に従ってインストールまたは展開します。macOS のセキュリティと隔離属性の扱いはリポジトリの [README.md](./README.md) を参照してください。

## 設定

初回は次を実施します。

1. サイドバー **設定** を開く。  
2. **Anthropic / OpenAI / 互換エンドポイント** のキー、モデル、Base URL（該当時）を設定。  
3. 保存。設定は `~/.squid/config.json` に書き込まれます。

任意：チャネル関連、UI 設定など（実際の設定ページのグループに準拠）。

## タスクとセッション

### タスクモード

| モード | 向いている用途 |
|--------|----------------|
| Ask | 相談、読み取り中心の分析、ワークスペースへの変更は最小限 |
| Craft | ツールの自動実行が必要。ファイルの作成・変更があり得る |
| Plan | 複雑な依頼：まず計画や手順を出し、確認しながら進める |

### 作成の流れ（概要）

1. **新規タスク** または同等の入口を選ぶ。  
2. モード、モデル、**作業ディレクトリ**（必須で信頼できるパス）を選ぶ。  
3. 任意：スキル、エキスパート。  
4. 自然言語で指示を送る。

ワークスペース外のパスは多くの場合サンドボックスで拒否されます。システムの機密ディレクトリをワークスペースにしないでください。

## スキル

- プリセットとインストール済みスキルをタスク作成または設定から選択。  
- 内容は `~/.squid/skills/` に保存（ディレクトリまたは単一ファイルをローダがサポート）。  
- Tencent SkillHub など外部ソースのインストールとメタデータは [tencent-skillhub.md](./tencent-skillhub.md) を参照。

## エキスパート

複数のロールテンプレートで回答のスタイルと専門境界を調整します。エキスパートセンターで確認・切り替え。カスタムエキスパートはバージョンによります。

## スケジュールタスク

1. **スケジュールタスク** ページを開く。  
2. 新規：Cron 式、トリガー後にモデルへ渡す内容、その他オプションを入力。  
3. 有効化後も **アプリ起動中のみ** スケジュールされます。終了すると停止。

日報要約やリポジトリ巡回などのテンプレートが提供されていれば、新規ウィザードから選択できます。

## チャネル（Channel）

- **WebUI**：メインのチャットとタスク。組み込みチャネルで実行エンジンと通信。  
- **拡張チャネル**：Feishu、Telegram、WeChat 個人アカウントなど。リポジトリの `extensions/` とユーザーの `~/.squid/extensions/` に配置し、`channel-plugin.json` で宣言。有効化とフォーム設定は [channel-extensions.md](./channel-extensions.md) を参照。

Feishu ではオープンプラットフォームアプリ、イベント購読（長接続または Webhook）、ローカルの `~/.squid/feishu-channel.json` などが必要です。HTTP コールバックではイベント URL が本機から到達可能である必要があります。詳細は [QUICK_START.md](./QUICK_START.md) と `extensions/feishu` の README を参照してください。

## メモリ

長期メモリは専用画面で閲覧・編集できます。保存場所は `~/.squid` 配下で実装に依存します。テストでは環境変数でメモリディレクトリを上書きできます（開発者ドキュメント参照）。

## Claw とローカル API（上級）

- メインデスクトップには **ローカル HTTP API**（既定では UI と同機）が内蔵され、タスク実行やストリーム出力などに使われます。未強化のまま公網に晒さないでください。  
- `src/claw` に Claw 関連 HTTP の実装があります。デスクトップ既定起動で有効かは `src/bun/index.ts` の現行ロジックに従います。リモート呼び出し、トークン、ルートはコードとテストを参照。

## データとバックアップ

| パス | 内容 |
|------|------|
| `~/.squid/config.json` | メイン設定とモデルキー等 |
| `~/.squid/skills/` | スキル |
| `~/.squid/channel-extensions.json` | ユーザー側チャネル拡張の有効化など |
| `~/.squid/extensions/` | ユーザー拡張のルートの一つ |

`~/.squid` 全体を定期的にバックアップしてください。秘密鍵をバージョン管理にコミットしないでください。

## よくある質問

**既定モデルの切り替えは？**  
設定で既定を変更するか、個別のタスク作成時に上書きします。

**ワークスペース外のファイルにアクセスできる？**  
既定では不可。サンドボックスと権限ルールに依存します。

**アンインストールや移行は？**  
アプリ終了後に `~/.squid` をバックアップまたは削除。移行時は新しいマシンに同ディレクトリを復元しアプリを再インストールします。
