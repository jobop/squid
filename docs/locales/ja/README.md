# squid

squid はローカルで動作する AI デスクトップワークベンチです。アプリウィンドウ内でモデルと対話し、複数セッションを管理し、セッションごとに作業ディレクトリを指定できます。設定から Anthropic、OpenAI、または互換 API に接続できます。スキル、メモリ、スケジュールタスク、および Feishu（Lark）、Telegram、WeChat などのチャネル機能（アプリ内で有効化・設定が必要）は、同じタスクパイプライン上で動作します。設定とデータは既定で **`~/.squid`** に保存されます。

**バージョン**：0.1.0  
**ライセンス**：MIT

---

## できること

- **マルチセッションのチャット**：スレッド単位で会話を整理し、セッションに作業ディレクトリを紐付け、特定プロジェクトに関する質問や、許可範囲内のファイル読み書きに利用できます。
- **タスクモード**：画面から **Ask**（相談・読み取り中心）、**Craft**（ツールチェーン実行）、**Plan**（計画・分解）など、目的に合ったモードを選択します。詳細はアプリ内の説明に従ってください。
- **モデルと API キー**：**設定**で API Key、モデル名、カスタムエンドポイントを入力します。キーはローカルのみに保存され、リポジトリには含まれません。
- **スキル**：アプリ内でスキルを閲覧・インストール・管理します（Tencent SkillHub などのソースを含む）。インストール済みは `~/.squid/skills` にあります。
- **エキスパートとメモリ**：プリセットまたはカスタムの「エキスパート」でスタイルと境界を調整します。長期メモリは別画面で確認・管理できます。
- **スケジュールタスク**：Cron 式でローカルに定期実行し、指定内容をモデルに渡し、実行履歴を残します。
- **チャネル**：メイン UI に加え、Feishu、Telegram、WeChat 個人アカウントなどの拡張チャネルを有効化できます（チャネル設定で構成が必要です。一部は追加ログインや Webhook が必要。詳細は `docs` および各チャネル向け資料を参照）。

## インストールと起動

**ソースから実行（開発者または自前ビルド）**

- **Node.js**（推奨 22 LTS）と **npm** が必要です。デスクトップシェルは **Electrobun** に依存し、macOS 14+、Windows 11+、および公式が案内する Linux 環境をサポートします。
- リポジトリをクローンしたうえで、プロジェクトルートで次を実行します。

```bash
cd squid
npm install
npm run dev
```

**配布パッケージ**

- GitHub Release などにインストーラが用意されている場合は、OS 向けの成果物をダウンロードしてインストールまたは展開して実行します。macOS で未署名・未公証のビルドは初回起動時にブロックされることがあり、「プライバシーとセキュリティ」から必要に応じて許可してください。

## はじめて使うとき

1. **設定**を開き、モデルおよび（必要なら）チャネルを保存します。  
2. チャット領域で**作業ディレクトリを選択**します（信頼できないパスをワークスペースにしないでください）。  
3. **新規セッション**を作成し、短い依頼で試します。自動化が必要になったらスキル、スケジュール、チャネルを試してください。

画面と手順の詳細は **[./QUICK_START.md](./QUICK_START.md)** と **[./user-guide.md](./user-guide.md)** を参照してください。  
多言語（中/英/日/露/伊/仏/独）は **[../../index.html](../../index.html)** からドキュメント入口に進み、言語を切り替えられます。

## データの保存場所

| 場所 | 意味 |
|------|------|
| `~/.squid/config.json` | モデル API キー、UI、一部機能スイッチなどのメイン設定 |
| `~/.squid/skills/` | インストール済みスキル |
| `~/.squid` 配下のその他 JSON | チャネル拡張、メモリなどモジュール別の設定・データ（利用に応じて生成） |

このディレクトリはユーザー側でバックアップしてください。公開リポジトリにキーをコミットしないでください。WeChat 個人アカウントなどの拡張では、端末で **`npm run weixin-personal:login`** を実行する場合があります（ソースから拡張を保守するときのみ）。詳細は拡張のドキュメントに従ってください。

## セキュリティ上の注意

- ファイルやコマンド系の能力は**作業ディレクトリ**と組み込みルールで制限されます。システムの機密ディレクトリを既定のワークスペースにしないでください。  
- アプリは UI とメインプロセス通信用のローカルサービスを提供します。通常利用では LAN やインターネットに意図的に公開しません。ポート転送やリバースプロキシを自前で行う場合は、認証とアクセス制御を必ず行ってください。

## ソースから開発に参加するとき（要点）

squid は **Electrobun** を採用しています。メインプロセスとローカルサービスは Bun 側、UI はシステム WebView です。**クローンしたリポジトリルート**で開発し、同梱のチャネル拡張を読み込む必要がある場合は、環境変数 **`SQUID_ROOT`** をそのリポジトリルートに向けます（`config/channel-extensions.json` を発見できるようにするため）。インストールパッケージ利用時は通常不要です。モジュール分割、拡張、ツールの約束事は **[./developer-guide.md](./developer-guide.md)** と **[./tool-development-guide.md](./tool-development-guide.md)** を参照してください。
