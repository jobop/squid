# squid クイックスタート

本稿はエンドユーザー向けです。ローカルに squid を入れ、モデル設定後すぐに対話とタスクを始められます。アーキテクチャと拡張は [developer-guide.md](./developer-guide.md) を参照してください。

## プロダクトの位置づけ

squid は **ローカルで動作する AI デスクトップワークベンチ** で、次の用途に適します。

- 指定した作業ディレクトリ上でのコード閲覧・レビュー・軽量編集（タスクモードとサンドボックスの制約下）
- スキルとエキスパート設定を組み合わせた文書整理、検索、構造化出力
- Cron 式によるローカルでのスケジュール実行
- 必要に応じて Feishu、Telegram、WeChat などの **チャネル拡張** を有効化（別途設定。[channel-extensions.md](./channel-extensions.md)）

## 環境と起動

**ソースから実行（開発者向け推奨）**

- Node.js 22 LTS 推奨、npm。デスクトップシェルは Electrobun に依存（`npm run dev` 実行時にプラットフォーム向け CLI を準備）。
- プロジェクトルートに **`electrobun.config.ts`** が必要です（Electrobun はこのファイル名のみ解釈します）。

```bash
cd squid
npm install
npm run dev
```

**配布パッケージ**

GitHub Release などからビルド済みインストーラを入手した場合は、各プラットフォームの手順に従ってください。macOS で未署名・未公証のビルドは、初回は Control キーを押しながら開くなどして許可が必要なことがあります。詳細はリポジトリルートの [README.md](./README.md) を参照してください。

## API キーの設定

少なくとも 1 つのモデルプロバイダを設定します（アプリ内 **設定** で保存。キーはローカルの `~/.squid/config.json` に書き込まれます）。

| プロバイダ | 説明 |
|------------|------|
| Anthropic | [Anthropic Console](https://console.anthropic.com/) で API Key を作成 |
| OpenAI | [OpenAI Platform](https://platform.openai.com/) で API Key を作成 |
| 互換エンドポイント | 設定でカスタム Base URL とモデル名を指定（アプリが利用するプロトコルと互換であること） |

## 初回の流れ

1. アプリ起動後、サイドバーの **「設定」** を開き、モデルと関連オプションを保存します。  
2. チャットまたはタスク画面で **作業ディレクトリを選択** します（信頼できないパスをルートにしないでください）。  
3. **新規セッションまたはタスク**を作成し、モードを選びます。  
   - **Ask**：読み取りと分析中心。既定ではファイルを積極的に書き換えません（バージョンによる）。  
   - **Craft**：ツールチェーンの自動実行が可能で、ワークスペース内のファイルを変更し得ます。  
   - **Plan**：計画と手順の提示に適した複雑な依頼向け。  
4. 必要に応じて **スキル** または **エキスパート** を選択します。

## チャネルと Feishu（任意）

- サイドバー **「チャネル」** で組み込み WebUI と各拡張の状態を確認できます。  
- Feishu の実装はリポジトリの `extensions/feishu/` にあります。`config/channel-extensions.json` で既定有効かはリポジトリ方針に依存します。ユーザー側の有効リストは `~/.squid/channel-extensions.json` に記述できます。  
- 個人または第三者の拡張は `~/.squid/extensions/<ディレクトリ>/` に置けます。詳細は [channel-extensions.md](./channel-extensions.md) を参照してください。

Feishu ボットの作成、長接続と Webhook、`~/.squid/feishu-channel.json` の各フィールドは、アプリ内の文言および [user-guide.md](./user-guide.md) を参照してください。

## よくあるタスク例

**コードレビュー（Ask）**

```text
モード：Ask
作業ディレクトリ：<あなたのプロジェクトパス>
指示：src 配下の主要モジュールの責務を整理し、可読性と明らかな欠陥について提案してください。
```

**文書の一括処理（Craft）**

```text
モード：Craft
作業ディレクトリ：<プロジェクトパス>
指示：指定ディレクトリ内の公開 API 向けに Markdown 説明のドラフトを生成してください。
```

**スケジュールタスク**

**スケジュールタスク** 画面で新規エントリを作成し、Cron とトリガー後にモデルへ渡す内容を入力します。アプリが起動していない間はスケジュールは動きません。

## スキルとエキスパート

- **スキル**：画面からインストール済みスキルを選択。ファイルは `~/.squid/skills/`（SkillHub などからのインストール結果を含む）。  
- **エキスパート**：システム上の役割と境界を調整。**エキスパート** 関連ページで組み込みとカスタムを管理します。

## よくある質問

**キーはローカルのみですか？**  
はい。設定とキーは Git にコミットせず、`~/.squid` をバックアップしてください。

**タスクはファイルを変更しますか？**  
モードとツール方針によります。Ask は読み取り寄り、Craft は書き込みの可能性、Plan は多くの場合まず説明します。画面の案内を優先してください。

**作業ディレクトリの境界は？**  
ファイル系ツールは多くの場合、セッションに紐付いた作業ディレクトリ内に制限されます。権限とサンドボックス実装を参照してください。

**実行中のタスクを止めるには？**  
タスクまたはセッション画面の停止／中断コントロールを使用します（ラベルは UI に準拠）。

**アプリを閉じたあともスケジュールは動きますか？**  
いいえ。スケジュールはアプリプロセスが生存していることが前提です。
